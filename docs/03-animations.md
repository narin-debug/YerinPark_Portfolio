# 애니메이션 스펙

## 1. 가로 스크롤 갤러리 (2번 섹션에 사용)
- 세로 스크롤이 섹션 진입 시 가로 이동으로 변환되는 방식 (GSAP ScrollTrigger
  pin + horizontal translate)
- 이미지마다 캡션(eyebrow 텍스트) 포함
- 섹션을 통과하는 데 필요한 세로 스크롤 분량은 이미지 개수에 비례하여 조정

## 2. 패럴랙스 (2번, 5번 섹션에 사용)
- 이미지/요소마다 스크롤 속도를 다르게 주어 입체감 형성
- 강도: 은은한 수준(과하지 않게)

## 3. 텍스트 라인 리빌 (히어로, 소개, 푸터 등 여러 곳에 사용)
- 텍스트를 줄 단위로 분리
- 각 줄 옆에서 색상 하이라이트(라임 컬러)가 채워지듯 지나가는 효과
- 스크롤에 반응하여 뷰포트 진입 시 재생

## 4. 프로젝트 카드 호버 (4번 섹션에 사용)
- 카드 프레임: 모서리 한쪽이 잘린 방패 형태의 커스텀 SVG path
- 기본 상태: 무채색/기본 컬러 스트로크 프레임 + 기본 썸네일
- 호버 상태:
  - 썸네일이 다른 이미지(또는 GIF/짧은 영상)로 크로스페이드
  - 프레임 테두리가 라임 컬러로 옆에서부터 그려지듯 나타남
    (`stroke-dasharray` / `stroke-dashoffset` 트랜지션)
- 프로토타입 SVG path 예시(407x411 viewBox 기준):
  ```
  M8 1h390.89a7 7 0 0 1 7 7v356.983a7 7 0 0 1-7 7H263.329a23.999 23.999 0
  0 0-18.766 9.038l-16.499 20.694A21.999 21.999 0 0 1 210.862 410H8a7 7 0
  0 1-7-7V8a7 7 0 0 1 7-7Z
  ```
- 참고: 원본 사이트에는 이 프레임과 연동된 Rive 벡터 캐릭터 애니메이션이
  스크롤에 반응해 별도로 존재했으나, 본 프로젝트에서는 사용하지 않고 위의
  호버 기반 SVG 리빌로 대체함(대체 방식 (b) 채택).
